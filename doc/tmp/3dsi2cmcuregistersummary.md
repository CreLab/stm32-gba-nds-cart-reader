# 3DS I2C MCU Register Summary


**Bus/Device 1:4Ah - MCU**
The McuIndex is usually automatically incremented after each data byte
(eg. allowing to read the two Version bytes in one pass).
However, there are a few arrays (at McuIndex=29h,2Dh,4Fh,61h,7Fh), which
will increment the ArrayIndex instead of the McuIndex. This happens only
if the index value (in the I2C command) was pointing to the array (eg.
writing 5 bytes to index 29h will update the Power LED Array. But,
writing 5 bytes to index 28h will update McuIndex 28h,29h,2Ah,2Bh,2Ch;
without getting stuck at 29h, and thus updating only the 1st byte of the
Power LED Array).
The extra-special case is using index 60h in the I2C command (it will
increment once, then stay at index 61h).

**Blurb**
On old versions of MCU_FIRM none of the invalid registers are masked
away by the read handler function, but are still read-only. Newer
MCU_FIRM versions return the hardcoded value FF instead.
Reportedly register 3Bh (RTC Alarm month) \"could be used on very ol
MCU_FIRM versions to upload MCU firmware if some conditions (?) are
met\", uh?.
